# Campaign Strategy OS — Claude 리뷰 브리프

## 리뷰 목적

이 브랜치는 Campaign Strategy OS v1 프로토타입의 제품 논리, 예시 품질, 카드 구조, 데이터 안전성을 검토하기 위한 공개 리뷰본이다. 구현을 수정하거나 배포하지 말고 먼저 문제점과 개선 제안을 보고한다.

## 제품의 역할

- 마케팅 프레임워크 자체를 수록하는 도구가 아니다.
- 프레임워크로 생각한 결과를 브리프와 실행 지침으로 정리하고, 기획과 실행 사이의 누락을 점검하는 도구다.
- 프로젝트 관리, 담당자 배정, 진행률, 알림, Notion 연동은 현재 범위가 아니다.

## 핵심 흐름

1. 작업 유형과 템플릿 선택
2. 질문과 카드를 이용해 생각 전개
3. 대표안·후보·제외안을 구분해 선택 근거 보존
4. 전략 정리
5. 목표 → 전략 → 활동 → 다음 행동 → 성공 신호 연결 점검
6. 최종 브리프 생성

## 이번 반영 사항

- 작업 유형·템플릿·예시 카드의 상단과 하단 기준선을 정리했다.
- 정보량이 다른 모든 카드를 같은 높이로 강제하지 않고, 같은 카드 계열과 같은 행에서만 정렬감을 맞췄다.
- 실제 제안 구조를 참고한 예시는 브랜드·제품·캐릭터·인물·카피·수치를 익명화하고 복수 자료와 새 작성 내용을 조합했다.
- 예시를 대표안·대안·제외안으로 구분하고, 제외 이유를 표시했다.
- 활동 카드에 활동 목적, 활동 수단, 다음 행동, 성공 신호, 연결 전략을 구체화했다.
- 논리 연결 화면에서 전략→활동→다음 행동→성공 신호를 한 흐름으로 확인할 수 있게 했다.
- 프로젝트 저장 스키마는 `SCHEMA_VERSION = 3`을 유지했다. 예시 설명용 메타데이터는 복제 프로젝트에 저장하지 않는다.

## 공개본에서 의도적으로 제외한 자료

- 원본 제안서 PDF
- 원본 파일명과 예시 간 대응표
- 내부 출처 추적 문서 `docs/example-source-map.md`

앱에 포함된 사례는 원본 문장이나 식별 가능한 고유 정보를 복사한 것이 아니라, 의사결정 구조를 익명 재구성한 예시다. 리뷰에서는 코드에 남은 식별 가능성이나 과도하게 구체적인 흔적이 없는지도 확인한다.

## 주요 파일

- `src/App.jsx`: 템플릿, 예시, 보드, 전략 정리, 연결 점검, 최종 브리프
- `src/index.css`: 화면 스타일과 반응형 보완
- `scripts/verify-project-data.mjs`: 템플릿·예시·마이그레이션·논리 연결 검증
- `README.md`: 실행 방법과 제품 개요

## 사전 검증 결과

- 데이터 검증 통과
- Vite 프로덕션 빌드 통과
- 데스크톱 1440px에서 예시 카드 높이와 하단 버튼 기준선 일치
- 모바일 390px에서 예시 모달·작업 보드·논리 연결 화면의 본문 가로 넘침 없음
- F&B SNS 예시 복제 후 활동→전략→다음 행동→성공 신호 연결 확인
- 브라우저 콘솔 error·warning 없음

## Claude 리뷰 요청

다음 순서로 검토하고, 코드를 수정하거나 배포하지 말고 리뷰 결과만 제시한다.

1. 이 도구가 실제 마케팅 기획과 실행 점검에 도움이 되는가
2. 전략 브리프·전략 전개안·실행기획안의 역할과 순서가 명확한가
3. 카드 구조가 의사결정을 돕는가, 입력 부담만 키우는가
4. 예시가 구체적이면서도 특정 브랜드나 원본을 식별하기 어렵게 구성됐는가
5. 대표안·대안·제외안이 목표와 전략의 차이를 실제로 설명하는가
6. 논리 연결 점검의 규칙에 허위 양성 또는 누락 가능성이 있는가
7. 기존 v2 데이터 마이그레이션과 v3 optional backfill이 안전한가
8. 접근성, 모바일 사용성, 저장·가져오기, XSS·데이터 손실 측면의 문제는 없는가
9. 현재 버전에서 반드시 고칠 항목과 이후 버전으로 미뤄도 되는 항목은 무엇인가

리뷰 결과는 `치명적 문제`, `중요 개선`, `선택 개선`, `제품 전체 평가`로 나누고, 가능한 경우 파일과 코드 위치를 근거로 든다.

## 작업 제한

- 배포하지 않는다.
- PR을 생성하지 않는다.
- 사용자 승인 없이 코드를 수정하지 않는다.
- 원본 자료를 추정하거나 특정 브랜드를 복원하려 하지 않는다.
